# 360° Gallery & Viewer

A browser-based gallery and viewer for 360° equirectangular photos and videos.
No server, no build step, no installation — open the HTML file and go.

Built to work with footage from the **Insta360 X4** and other equirectangular cameras
(GoPro Max, Ricoh Theta, etc.).

---

## Files

| File | Version | Description |
|---|---|---|
| `360-gallery.html` | v1.0.2 | Multi-item gallery with thumbnails, filter, sort, and fullscreen viewer |
| `360-viewer.html` | v1.0.2 | Single-file standalone viewer for one photo or video at a time |

Past releases are archived in the `versions/` folder.

---

## Features

### Gallery (`360-gallery.html`)
- Drag-and-drop or file picker import — select multiple files at once
- Equirectangular centre-crop thumbnails generated in-browser (no WebGL required)
- Video thumbnails captured from the first second of footage
- Filter by all / photos / videos
- Sort by name (A–Z, Z–A), date added, or type
- Fullscreen 360° viewer on click
- Keyboard navigation: `←` `→` to move between items, `Esc` to close
- Swipe navigation on mobile
- Play / pause, volume / mute controls for video

### Viewer (`360-viewer.html`)
- Drag-and-drop or file picker for a single photo or video
- Full pan, zoom, and autorotate controls
- Play / pause, volume / mute controls for video
- Detailed error reporting panel

## Usage

### On desktop
1. Download `360-gallery.html`
2. Open it directly in Chrome, Edge, Firefox, or Safari
3. Drop your 360° files onto the page or click **Add media**

### On mobile
Host on GitHub Pages (see below) and open the URL in any mobile browser.

### On GitHub Pages
Once enabled, the gallery is available at:
```
https://wheelsalert.github.io/360-gallery/360-gallery.html
```

## Supported formats

| Type | Formats |
|---|---|
| Photos | `.jpg`, `.jpeg`, `.png`, `.webp` |
| Videos | `.mp4`, `.webm` |

All media must be **equirectangular projection** — the standard output format
from 360° cameras. Non-equirectangular images will load but will appear distorted.

## How it works

### Architecture
Both files are self-contained single-page apps with no build step.
Dependencies are loaded at runtime via ES Module imports from jsDelivr CDN.
An `importmap` in the HTML maps bare package names to CDN URLs so the browser
resolves them correctly.

### Thumbnail generation
Gallery thumbnails are generated by cropping the centre third of each
equirectangular image (33–66% width, 25–75% height). This region corresponds
to the "looking straight ahead" view and has the least spherical distortion.
The crop runs on a `<canvas>` element — no WebGL or server processing required.

For videos, the browser seeks to the first second of footage before capturing
the frame.

### Photo loading
PSV's internal texture loader uses Three.js `FileLoader`, which creates a
`Request` object with an `AbortSignal` attached. This object cannot be
transferred across browser contexts (`DataCloneError`). To work around this,
photos are pre-loaded into PSV's own `Cache` object before the viewer is
initialised, so `FileLoader` is never called for local files.

### Video loading
A single `<video>` element is created without a `crossOrigin` attribute and
kept permanently in the DOM. Blob URLs created from local files are assigned
to this element directly. This avoids the browser's URL safety check, which
rejects blob URLs when `crossOrigin="anonymous"` is set (the default behaviour
in PSV's internal video element creation).
